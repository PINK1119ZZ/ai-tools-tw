n8n 2.14 改了什麼：MCP 從「執行」到「建立」

如果你一直在用 n8n，你一定記得以前的痛點：即使設定好 n8n 的 MCP Server，Claude Code 或 Cursor 也只能「觸發執行」已存在的 Workflow，沒辦法幫你「新建」或「修改」工作流程。

這個限制在 n8n 2.14 正式終結。

n8n 2.14 Instance-Level MCP Server 新增兩個關鍵工具：
  • create_workflow_from_code：讓 AI 直接以 JSON 建立全新 Workflow
  • update_workflow：讓 AI 修改現有 Workflow 的節點、連線、設定

這代表什麼？舉個例子：你可以對 Claude Code 說「幫我建一個 n8n workflow，每天早上 9 點抓 Google Sheets 的新增訂單，發 Line Notify 給我」，Claude 會直接透過 MCP 在你的 n8n 實例建立這個 Workflow，連你打開瀏覽器都不需要。

技術架構：n8n MCP Server 如何運作

n8n 的 Instance-Level MCP 是一個架設在你 n8n 實例上的 MCP Server。Claude Code、Cursor、Codex CLI 等任何支援 MCP 協議的 AI 工具，都可以透過它與 n8n 溝通。

版本 MCP 功能 說明
n8n 2.12 以前 execute_workflow 只能觸發執行現有 workflow
n8n 2.13 + list_workflows, get_workflow 可以列出和讀取 workflow 資訊
n8n 2.14 🆕 + create_workflow_from_code
+ update_workflow		 AI 可直接建立與修改 workflow

加上原本就有的 execute_workflow、list_workflows、get_workflow，現在的 n8n MCP Server 已經是一個相當完整的「n8n 遠端控制介面」，AI Agent 幾乎可以做到所有你在 GUI 上能做的事。

5 分鐘設定教學：讓 Claude Code 連接你的 n8n

Step 1：確認 n8n 版本

# 如果是 npm 安裝
npm list n8n | grep n8n
# 如果是 Docker
docker exec -it n8n n8n --version

確保版本 ≥ 2.14.0。如果未更新：

# npm 升級
npm update -g n8n

# Docker 更新
docker pull n8nio/n8n:latest
docker-compose up -d

Step 2：取得 n8n API Key

  1. 登入你的 n8n 介面
  2. 右上角點個人圖示 → Settings
  3. 左側選 API
  4. 點 Create an API key
  5. 複製 API Key（只顯示一次！）

Step 3：在 Claude Code 設定 n8n MCP

# 編輯 Claude Code MCP 設定
# 路徑：~/.claude/claude_desktop_config.json

{
  "mcpServers": {
    "n8n": {
      "command": "npx",
      "args": ["-y", "@n8n/mcp-server"],
      "env": {
        "N8N_BASE_URL": "https://你的n8n網址",
        "N8N_API_KEY": "你的API_KEY"
      }
    }
  }
}

如果是自架在 DigitalOcean 或 Cloudways 的 n8n，把 N8N_BASE_URL 改為你的伺服器 IP 或網域。

Step 4：測試連線

在 Claude Code 輸入：

請列出我的所有 n8n workflow

如果 Claude 回傳你的 workflow 清單，代表連線成功！

n8n 2.14 其他重要新功能

除了 MCP 大升級，n8n 2.14 還帶來了幾個值得關注的更新：

MCP Workflow Tools 節點：現在可以在 workflow 內部使用 MCP 工具節點，意思是你的 n8n workflow 可以直接呼叫外部 MCP Server（例如讓 n8n 呼叫 GitHub MCP 或 Figma MCP）。AI 自動化的串聯深度大幅提升。
1Password External Secrets 整合：企業用戶可以把 n8n 的 API Key 存在 1Password Secrets Manager，不再需要把密鑰直接寫在 n8n 的環境變數裡，安全性大幅提升。
SSRF 防護強化：n8n 2.14 加強了 Server-Side Request Forgery 防護，自架版用戶不再需要擔心 workflow 被惡意利用來探測內網。
Workflow History 壓縮：n8n 現在會自動壓縮舊版本的 workflow 歷史記錄，避免資料庫膨脹。自架用戶的維護壓力減輕。

AI Builder 改進

n8n 的視覺化 AI Agent Builder 修正了幾個影響使用者體驗的問題：

  • AI Builder 評估（Eval）結果顯示錯誤修正
  • Chat Memory 節點的記憶保留問題修正
  • AI 節點回應格式解析更穩定
