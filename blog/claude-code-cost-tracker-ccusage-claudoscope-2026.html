📊 ccusage：4800+ ⭐ 的病毒級成本追蹤 CLI

這是什麼？

ccusage 是一款開源命令列工具，讀取 Claude Code 在本機儲存的 JSONL 工作階段檔案（位於 ~/.claude/projects/），分析你的 Token 消耗量並換算成 USD 費用。不需要連線、不需要 API Key、不傳送任何資料到外部伺服器——一切都在你自己的機器上運算。

它在 2026 年初爆紅，GitHub 在短短幾週內衝到 4800+ ⭐，YouTube 甚至有影片專門介紹「那些每月燒 $8000+ token 的人的 ccusage 報告」。核心吸引力很簡單：你終於可以看到 Claude Code 帳單的真相。

💡 ccusage 如何運作：
Claude Code 在每次對話時，會將完整的 Token 使用紀錄寫入本機 JSONL 檔案（位於 ~/.claude/projects/<project-hash>/）。ccusage 解析這些檔案，對照 Anthropic 公開的定價表（Opus、Sonnet、Haiku 各不同），計算出你的實際或等效費用。完全本地執行，無需 API 呼叫。

ccusage 核心功能

  • 📅 每日報告（Daily）：按日期彙整 Token 消耗與費用，一眼看出哪天最燒錢
  • 📆 每週報告（Weekly）：以週為單位追蹤使用模式，可自訂週起始日
  • 📈 每月報告（Monthly）：月度費用分析，和訂閱費用比較，算出實際划算程度
  • 💬 Session 報告：每個對話工作階段的費用細分，找出最耗費的任務
  • ⏰ 5 小時 Block 監控：追蹤 Claude 的 5 小時計費視窗，避免突破限額
  • 🤖 Model 追蹤：分別顯示 Opus / Sonnet / Haiku 各自的消耗
  • 🔄 Cache 支援：分別追蹤 cache creation 和 cache read Token（省錢關鍵）
  • 📤 JSON 輸出：匯出結構化資料，方便自建 Dashboard 或串接其他工具
  • 🌐 離線模式：使用預建定價資料，不需要網路連線

⚙️ ccusage 安裝與使用教學（5 分鐘上手）

安裝方式

ccusage 透過 npm 安裝，前提是你已經有 Node.js 環境（Claude Code 需要，所以你應該已有）：

# 全域安裝（推薦） npm install -g ccusage # 或使用 npx 直接執行（不用安裝） npx ccusage

基本指令速查

# 每日費用報告（最常用） ccusage daily # 每月費用報告 ccusage monthly # 每週費用報告 ccusage weekly # 每個 Session 的費用細分 ccusage session # 即時監控（即時更新目前 Session） ccusage monitor # 匯出 JSON 格式 ccusage daily --json # 查看特定日期範圍 ccusage daily --since 2026-04-01 --until 2026-04-30

🔭 Claudoscope：macOS 原生的 Session 分析 App

這是什麼？

Claudoscope 是一款 macOS 原生 Menu Bar App，用 Swift/SwiftUI 打造，解決「我有幾千個 Claude Code Sessions，完全找不到」的問題。它讀取 ~/.claude/projects/ 中的 JSONL 工作階段檔案，提供圖形介面讓你瀏覽、搜尋、分析每個對話。

跟 ccusage 的差異在定位：ccusage 是聚焦在「費用分析」，Claudoscope 更像是一個「Claude Code Session 管理器」——你可以搜尋特定對話內容、查看工具調用記錄、偵測是否有 API Key 暴露的安全問題。

Claudoscope 核心功能

  • 🗂️ Session 瀏覽器：圖形介面瀏覽所有歷史 Claude Code 工作階段，按日期/專案分組
  • 🔍 全文搜尋：搜尋所有 Sessions 的完整對話內容（包含工具輸出），快速找到你要的程式碼片段
  • 💰 Token 與費用分析：每個 Session 的 Token 消耗量和費用估算
  • 🔐 即時 Secret 偵測：掃描 Sessions 中是否有暴露的 API Key、密碼、Token（44 條規則）
  • 🏥 Config 健康檢查：分析 Claude Code 設定檔，提供 44 條 linting 規則建議
  • 🧩 工具調用顯示：完整顯示每次 Tool Call 的輸入輸出，debug 神器
  • 🧠 Thinking Block 支援：顯示 Claude Extended Thinking 的思考過程（如有開啟）
  • ⚙️ Plugin/Skill/Hook 視覺化：呈現 Claude Code 的 Agent Skills、Memory、Plugin 設定
  • 🔒 完全離線：零 API 呼叫、零遙測、完全本地執行
📦 技術規格：
  • 語言：Swift / SwiftUI
  • 安裝：Homebrew Cask
  • License：MIT 開源
  • 平台：macOS only（macOS 13+ Ventura 以上）
  • GitHub：cordwainersmith/claudoscope

🛠️ Claudoscope 安裝與功能教學

Homebrew 安裝（推薦）

# 加入 Claudoscope 的 Homebrew tap brew tap cordwainersmith/tap # 安裝 Claudoscope（作為 macOS App） brew install --cask claudoscope # 安裝完成後，在 Applications 資料夾找到並開啟 # 或直接從 Launchpad 搜尋 Claudoscope

安裝完成後，Claudoscope 會出現在 macOS Menu Bar（右上角）。點擊圖示即可開啟 App，它會自動掃描 ~/.claude/projects/ 下的所有 Sessions，不需要任何設定。

第一次開啟會看到：
  1. 左側面板：按專案和日期分組的所有 Session 列表
  2. 右側主區：選取 Session 後顯示完整對話，包含工具調用
  3. 頂部工具列：全文搜尋框、篩選器、Token 統計
  4. 底部 Status Bar：Token 總計、費用估算

最實用功能：全文搜尋

如果你的 Claude Code Sessions 累積了幾百甚至幾千個，「找那個兩週前解決 Django ORM 問題的對話」幾乎是不可能的任務。Claudoscope 的全文搜尋解決了這個問題：

# 在 Claudoscope 搜尋框輸入關鍵字： # - 搜尋程式碼片段：async def get_user # - 搜尋工具名稱：write_file # - 搜尋錯誤訊息：TypeError cannot read # - 搜尋你的提問：如何設定 nginx # 搜尋範圍涵蓋： # ✓ 對話內容（你的問題 + Claude 的回答） # ✓ 工具調用的輸入/輸出 # ✓ Claude 的 Thinking blocks（如有） # ✓ 系統提示詞（CLAUDE.md 等）

安全功能：Secret 即時偵測

這是 Claudoscope 最獨特的功能之一。它內建 44 條規則，掃描你的 Claude Code Sessions 中是否不小心貼上了 API Key、資料庫密碼、JWT Token 等敏感資訊。如果偵測到問題，會立即在介面上警告你。

對於台灣的 AI 開發者來說，這個功能特別重要——Claude Code 在協助你寫程式時，常常需要看到設定檔、環境變數等，稍不注意就可能把 AWS Key 或 Stripe Secret 暴露在 Session 記錄中。

💡 台灣開發者 Claude Code 省錢 5 招

有了 ccusage 的數據加持，你可以更精準地優化 Claude Code 使用方式。以下是根據台灣開發者社群整理的 5 個實用省錢技巧：

1. 善用 Prompt Caching（最重要！）

Prompt Cache 可以把重複出現的大型系統提示（如 CLAUDE.md、長篇上下文）快取起來，後續呼叫的 cache read 費率只有原始 input token 的 10%。從 ccusage 的報告中，你可以看到 "Cache Read Tokens" 欄位——數值越高，代表你越善用快取。

# 優化前（每次都重新輸入 5000 token 的系統提示）： Cost per day: $12.50 # 優化後（善用 prompt cache）： Cache Read Tokens: 4,500,000 Cost per day: $4.20 ← 省了 66%！

2. 用 ccusage 找出「費用怪獸」Session

執行 ccusage session 列出所有 Session 費用，你往往會發現有幾個 Session 佔了總費用的大多數。這些通常是超長對話、包含大型檔案的任務、或是讓 Claude 反覆讀同一份文件的任務。找到這些後，可以考慮把它們拆成較小的任務。

3. 依任務選擇合適模型

從 ccusage 的 model tracking 可以看出，Claude Opus 的費用是 Sonnet 的 5 倍。對於不需要最高智能的日常任務（如格式轉換、基本程式碼補全），明確指定使用 Sonnet 就好：

# 在 CLAUDE.md 中設定預設 model 策略： ## Model Selection Guide - Complex architecture decisions: Use Opus 4.6 - Regular coding tasks: Use Sonnet 4.6 (default) - Simple formatting/refactoring: Use Haiku 3.5

4. 訂閱 vs 按量計費的選擇

如果你用 ccusage 算出的「等效 API 費用」每月超過 $60，Claude Max ($100/月) 通常比 API 按量計費划算。如果每月等效費用低於 $20，Pro ($20/月) 就夠了；如果低於 $10，不訂閱直接用 API 反而更省。

5. 清理長 Session，從短 Context 開始

Claude Code 的 Token 費用和 Context Window 大小正相關。一個已經延續幾十個小時的超長 Session，每次呼叫都要傳送大量歷史記錄。定期用 /compact 指令整理對話、或開新 Session，可以顯著降低成本。ccusage 的 Monitor 模式會告訴你當前 Session 的 context 大小，超過一定門檻就考慮開新 Session。
