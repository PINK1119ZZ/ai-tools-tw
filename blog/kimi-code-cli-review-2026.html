Kimi Code CLI 是什麼？

2026 年 4 月，Moonshot AI 在發布 Kimi K2.6 模型的同時，推出了 Kimi Code CLI——一款以 K2.6 為底層模型的終端機 AI coding agent。它的定位非常明確：對標 Claude Code，但成本低 10 倍。

如果你用過 Claude Code，Kimi Code CLI 的操作邏輯幾乎一樣：在終端機輸入自然語言指令，AI 自動讀取程式碼、修改檔案、執行測試、提交 commit。差別在於背後的模型換成了 Kimi K2.6，而且支援最多 300 個子 Agent 並行，適合大型重構或多模組同時開發。

一句話定位：Kimi Code CLI = Claude Code 的平替版，適合預算有限但需要長時程 agentic coding 的台灣開發者。

核心功能與特色

1. Agent Swarm — 300 子 Agent 並行

Kimi Code CLI 最大的差異化功能是 Agent Swarm。一個主 orchestrator 可以同時派出最多 300 個子 Agent，各自負責不同模組或任務，最多協調 4,000 步驟。這讓它特別適合：

  • 大型 monorepo 重構（前後端同時進行）
  • 批次生成測試（每個模組一個 Agent）
  • 多語言文件同步翻譯
  • 平行爬蟲 + 資料清洗 pipeline

2. 原生多模態輸入

Kimi Code CLI 支援直接貼入截圖、設計稿、影片截幀作為輸入。你可以把 Figma 截圖丟給它，直接生成對應的 React 元件——這是 Claude Code 目前還不支援的功能。

3. 自動 MCP 發現

啟動時會自動掃描本機已安裝的 MCP Server（如 Playwright、Filesystem、GitHub），無需手動設定。這讓它能直接操控瀏覽器、讀寫檔案、呼叫 GitHub API。

4. IDE 整合

除了純終端機模式，Kimi Code CLI 也提供 VSCode、Cursor、Zed 的 extension，讓你在 IDE 內直接呼叫 Agent Swarm，不用切換視窗。

5. 長時程自主執行

K2.6 模型支援 12+ 小時的自主執行，中途不需要人工介入。適合設定好任務後讓它跑整晚，早上起來看結果。

5 分鐘安裝教學

方法一：npm 安裝（最簡單）

# 全局安裝 Kimi Code CLI
npm install -g @moonshot-ai/kimi-code

# 設定 API Key（從 kimi.com 取得）
export KIMI_API_KEY="your-api-key-here"

# 或加入 ~/.bashrc / ~/.zshrc
echo 'export KIMI_API_KEY="your-api-key"' >> ~/.zshrc
source ~/.zshrc

# 驗證安裝
kimi-code --version

方法二：直接使用 OpenAI 相容 API

Kimi K2.6 的 API 完全相容 OpenAI 格式，可以在任何支援 OpenAI API 的工具（包括 Claude Code）中切換使用：

# 在 Claude Code 中使用 Kimi K2.6 模型
claude --api-url https://api.moonshot.cn/v1 \
       --api-key $KIMI_API_KEY \
       --model kimi-k2-6

# 或設定環境變數
export ANTHROPIC_BASE_URL=https://api.moonshot.cn/v1
export ANTHROPIC_API_KEY=$KIMI_API_KEY

方法三：自架 vLLM（最省錢）

適合有 GPU 伺服器的開發者，K2.6 支援 INT4 量化，可在 4×A100 上自架：

# 使用 vLLM 部署（需要至少 4×A100 80GB）
pip install vllm

python -m vllm.entrypoints.openai.api_server \
  --model moonshot-ai/kimi-k2-6 \
  --quantization awq \
  --max-model-len 65536 \
  --tensor-parallel-size 4
台灣開發者推薦：自架的硬體成本很高，大多數人直接用官方 API（$0.60/$2.80 per MTok）就夠了。如果用 DigitalOcean GPU Droplet 自架，每月省下的費用需要約 3 個月才能回本。

VSCode Extension 設定

# 在 VSCode 安裝 Kimi Code 擴充功能
code --install-extension moonshot-ai.kimi-code

# 或在 VSCode 搜尋「Kimi Code」安裝
# 設定 settings.json：
{
  "kimiCode.apiKey": "your-api-key",
  "kimiCode.model": "kimi-k2-6",
  "kimiCode.maxAgents": 10  // 設定並行子 Agent 數量
}

總結：誰適合用 Kimi Code？

✅ 適合你的情況

  • Claude Code 帳單讓你心痛
  • 需要大規模批次重構
  • 有多模態輸入需求（截圖→程式碼）
  • 想自架模型控制成本
  • 需要 12+ 小時長時程自主執行
  • 預算有限的個人開發者或新創

❌ 可能不適合的情況

  • 需要超大 context（>256K tokens）
  • 已深度整合 Claude 生態系
  • 需要 IDE 內即時 Tab 補全
  • 企業合規要求使用特定廠商
  • 需要 Anthropic 的 Routines 功能

Kimi Code CLI 不是要取代 Claude Code，而是填補了「需要 agentic coding 但付不起 Claude 費用」的空缺。對台灣的個人開發者、新創團隊、自由工作者來說，它是目前 CP 值最高的終端機 coding agent。

最務實的策略：Kimi Code CLI 做批次重構和長時程任務，Cursor Composer 2 做日常 IDE 開發，Claude Code 留給需要超大 context 的複雜任務。三款工具互補，不是非此即彼。
